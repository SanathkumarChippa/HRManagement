# HR Management Capstone Project

## Project Overview
This is a comprehensive enterprise-grade HR Management System built using the **Clean Architecture** pattern and **.NET 8.0**. The application features an integrated ASP.NET Core MVC web portal and a secure REST API.

## Architecture
The system is divided into logical layers following Clean Architecture principles:
- **Domain**: Core entities (`Employee`, `Department`, `LeaveRequest`, etc.) and basic interfaces.
- **Application**: Business logic, CQRS handlers/services, FluentValidation rules, AutoMapper profiles, and interface contracts.
- **Infrastructure**: External services integration (`TokenService` for JWT generation, Email services, etc.).
- **Persistence**: Entity Framework Core implementation, `ApplicationDbContext`, database migrations, repository implementations, and data seeding.
- **API**: RESTful API endpoints secured by JWT Authentication.
- **Web (MVC)**: Server-side rendered UI utilizing Bootstrap 5.3, DataTables, Chart.js, SweetAlert2, and Toastr. It directly consumes the Application and Persistence layers (Architecture Option B).

## Database Setup & Migrations
The database leverages **SQL Server** and Entity Framework Core Code-First Migrations.

### Table Prefixing
All tables generated by the framework are dynamically prefixed with `kumarcapstone_` (e.g., `kumarcapstone_Employees`, `kumarcapstone_Departments`).

### Run Migrations
To set up or update the database schema, execute the following command from the repository root:
```bash
dotnet ef database update --project src/HRManagement.Persistence --startup-project src/HRManagement.Web
```

### Seed Data
The database automatically seeds the following upon initialization:
- **Roles**: Admin, HR Manager, Employee
- **Departments**: HR, IT, Finance, Sales, Marketing, Operations
- **Leave Types**: Casual Leave, Sick Leave, Paid Leave, Maternity Leave
- **Default Users**: 
  - Admin (`admin@hrmanagement.com` / `Admin@123`)
  - HR Manager (`hr@hrmanagement.com` / `HRManager@123`)

## Setup Instructions
1. **Prerequisites**: Ensure you have the .NET 8.0 SDK and SQL Server installed.
2. **Clone Repository**: Clone the code to your local machine.
3. **Database Connection String**: Update `appsettings.json` in both `HRManagement.API` and `HRManagement.Web` projects to point to your SQL Server instance.
4. **Restore Dependencies**: 
   ```bash
   dotnet restore
   ```
5. **Update Database**: 
   ```bash
   dotnet ef database update --project src/HRManagement.Persistence --startup-project src/HRManagement.Web
   ```
6. **Run Web Portal**:
   ```bash
   dotnet run --project src/HRManagement.Web
   ```
7. **Run API**:
   ```bash
   dotnet run --project src/HRManagement.API
   ```

## Unit Testing
Unit tests are implemented using **xUnit**, **Moq**, and **FluentAssertions**. The project boasts a suite of 20+ passing tests covering:
- Leave workflow logic and balance calculations
- Employee and Department validation rules
- JWT generation and security claims

To run the tests:
```bash
dotnet test
```

## Security & Authorization
- **JWT Authentication**: The API is secured using JSON Web Tokens (JWT).
- **Role-Based Policies**: The system enforces authorization via claims-based policies:
  - `AdminOnly`
  - `HRManagerOnly`
  - `EmployeeOnly`
  - `AdminOrHR`
- **Soft Delete**: Records are soft-deleted (`IsDeleted` flag) rather than permanently removed, utilizing Global Query Filters in EF Core.
- **Concurrency**: Optimistic concurrency is handled globally via `RowVersion` bytes on the base entity.
